# Shift Report And HandOff Protocol

## Purpose

Define the minimum content and communication pattern for safe shift-to-shift handoff.

## Required Elements

- patient identifiers within approved workflow context
- current condition and priority risks
- recent changes in status
- pending tests, orders, or tasks
- escalation concerns
- follow-up timing

## Communication Rule

- Use concise SBAR-style communication.
- Read back critical items when the situation is high risk.
- Document the handoff completion where policy requires it.
