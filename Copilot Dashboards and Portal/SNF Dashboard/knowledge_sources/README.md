# Knowledge Sources

This folder contains runtime knowledge files for SNF AI Dashboard.

Use these sources the way Microsoft Learn describes knowledge sources and file groups:

- give each source a clear name that tells the agent what the source covers
- keep each source focused on one domain, policy area, or workflow area
- write descriptions and instructions so the agent knows when to use the source
- prefer detailed, task-specific descriptions over vague labels
- keep instructions explicit about scope, routing, exclusions, and expected use
- use file groups when different roles, workflows, or policy contexts should route to different subsets of files
- give each file group its own clear name, description, and instructions so the agent can select the right knowledge boundary

Current focus:

- therapy and SNF documentation guidance
- clinical warning labels and disclaimers
- line-staff quick-start guidance
- data quality and lineage FAQ
- workflow and escalation principles
- file and image upload guidance
- clinical file processing boundaries
- clinical image review boundaries
- metric governance and data quality explanation
- research, IRB, and governed data-use boundaries

Guidelines:

- keep each source narrow, well named, and task-specific
- prefer official and facility-approved sources
- align source names, descriptions, and instructions to the actual user task they support
- avoid vague source labels, multi-purpose source descriptions, or hidden implementation caveats in source descriptions
- update review dates and ownership in the starter bundle manifests when content changes
