# QM Action Plan Standard Operating Procedure

## Purpose

This document defines the standard operating procedure for generating, reviewing, and executing Quality Measure action plans within the SimpleLTC QM Coach agent. All action plans follow the 7-30-90 day framework and require Human-in-the-Loop (HITL) approval before implementation.

## Authoritative Sources

- CMS Medicare Benefit Policy Manual, Pub. 100-02, Ch. 15, §220.2 (therapy coverage standards)
- Jimmo v. Sebelius Settlement Agreement (maintenance therapy coverage)
- CMS State Operations Manual, Appendix PP (F-tag guidance for survey compliance)
- CMS QAPI (Quality Assurance and Performance Improvement) requirements at 42 CFR §483.75

## Action Plan Framework: 7-30-90 Days

### 7-Day Actions (Immediate Quick Wins)
- **Purpose**: Address immediate safety risks and establish baseline interventions
- **Owner**: Therapy Director of Rehab (DoR) and Nursing Director of Nursing (DON)
- **Examples**:
  - Initiate safety huddles for residents flagged on the target QM
  - Complete targeted screenings (fall risk, pain, skin integrity) for at-risk residents
  - Review and update care plans for residents contributing to the numerator
  - Implement immediate environmental modifications (grab bars, lighting, bed alarms)
  - Begin restorative nursing program referrals for residents with ADL decline
- **Escalation**: If 7-day actions cannot be completed, escalate to Administrator and Regional Therapy Consultant within 48 hours

### 30-Day Actions (Workflow Redesign)
- **Purpose**: Address systemic process gaps identified in the driver analysis
- **Owner**: Interdisciplinary Team (IDT) led by DoR and DON
- **Examples**:
  - Redesign therapy-nursing handoff workflows for the target QM domain
  - Implement standardized assessment tools (e.g., Braden Scale for pressure ulcers, Morse Fall Scale for falls)
  - Create or update interdisciplinary care conference templates
  - Establish weekly QM monitoring dashboards using SimpleLTC exports
  - Train therapy and nursing staff on updated protocols
  - Initiate MDS coding accuracy audits for items driving the target QM
- **Escalation**: If 30-day actions are not on track, escalate to Regional VP of Clinical Services

### 90-Day Actions (Stabilization and Sustainability)
- **Purpose**: Confirm improvement trend and embed changes into standard operations
- **Owner**: Administrator with IDT oversight
- **Examples**:
  - Validate QM trend improvement using 3 consecutive monthly data points
  - Update Facility Assessment to reflect new clinical capabilities
  - Align MDS coding practices with updated clinical workflows
  - Conduct post-implementation audit of action plan effectiveness
  - Document lessons learned and update SOPs
  - Transition from active intervention to ongoing monitoring
- **Escalation**: If 90-day targets are not met, initiate formal QAPI project with root cause analysis

## Action Plan Template

Each action plan generated by the agent must include:

```

GENERATED BY: SimpleLTC QM Coach V2 (AI-generated decision support)
```

## Roles and Responsibilities

| Role | Responsibility |
|------|---------------|
| Director of Rehab (DoR) | Leads therapy-specific actions, coordinates with nursing, presents at IDT |
| Director of Nursing (DON) | Leads nursing-specific actions, ensures MDS accuracy, coordinates care plan updates |
| Administrator | Approves resource allocation, monitors 90-day outcomes, escalates to regional |
| Regional Therapy Consultant | Reviews action plans for regulatory alignment, provides clinical coaching |
| MDS Coordinator | Validates coding accuracy for QM-driving MDS items |
| IDT Members | Execute assigned actions, report progress at weekly huddles |

## Escalation Rules

| Trigger | Escalation Path | Timeline |
|---------|----------------|----------|
| 7-day actions incomplete | DoR → Administrator → Regional Consultant | Within 48 hours of missed deadline |
| 30-day actions off track | IDT → Administrator → Regional VP | Within 1 week of identified delay |
| 90-day targets not met | Administrator → Regional VP → QAPI Committee | Within 2 weeks of quarter close |
| High-concern QM with no improvement after 90 days | Formal QAPI project initiation | Immediate |
| Survey-related QM deficiency | Administrator → Corporate Compliance → Legal | Within 24 hours |

## HITL Approval Requirements

All action plans generated by the agent are advisory until approved:
1. The agent generates the plan and presents it to the user
2. The user reviews the plan for clinical accuracy and feasibility
3. The user provides HITL approval (name, timestamp, decision)
4. Only after approval should the plan be shared with the IDT for execution
5. The agent stores the approval decision in Global.HITLApprovalDecision

## How the Agent Uses This Document

When generating action plans:
1. Follow the 7-30-90 framework exactly
2. Assign owners from the roles table based on the action type
3. Map each action to a specific CMS reference or ONE Clinical Protocol
4. Apply escalation rules when generating follow-up recommendations
5. Always require HITL approval before marking any plan as final
6. Never generate actions that contradict licensed clinician judgment
