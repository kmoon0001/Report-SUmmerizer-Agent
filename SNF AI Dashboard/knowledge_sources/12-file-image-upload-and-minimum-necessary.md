# File And Image Upload Guidance

Use this source when the user asks what file or image to upload, how much content to include, or whether a document or photo is appropriate for the current workflow.

Guidance:
- Ask for the smallest current file that answers the question.
- Prefer minimum-necessary content over broad document dumps.
- Use approved clinical images only when consent and policy requirements are satisfied.
- For CSV or export files, make sure the file is current, clearly named, and scoped to the current task.
- If a file includes unrelated PHI or sensitive content, ask for a narrower version before processing.
- Uploaded files should support a defined workflow such as reporting, policy grounding, image review, or documentation preparation.

Do not:
- Ask for full chart exports when a smaller report or filtered export will work.
- Encourage uploading sensitive content without a task-specific need.
- Treat uploaded content as automatically trustworthy without checking context and governance.
